# SEO Checklist

Cette checklist regroupe les points essentiels à vérifier pour garantir une base SEO solide.  
Elle couvre **uniquement les aspects stratégiques** : contenu, structure, maillage interne, indexation, données structurées, sémantique.

Pour les optimisations techniques (rendu, JS, images, caching, Core Web Vitals), voir la section **Références techniques** en bas de page.

---

## 📚 1. Contenu

### ✔️ Intention & pertinence

- [ ] Une page = une intention claire
- [ ] Contenu aligné avec l’intention
- [ ] Pas de cannibalisation entre pages
- [ ] Contenu utile, précis, complet

### ✔️ Structure éditoriale

- [ ] Un seul `<h1>`
- [ ] Hiérarchie correcte des titres
- [ ] Paragraphes courts
- [ ] Listes et tableaux utilisés si pertinents

### ✔️ Qualité & profondeur

- [ ] Contenu riche et expert
- [ ] Exemples, cas d’usage, explications
- [ ] Variations naturelles du vocabulaire
- [ ] Pas de keyword stuffing

---

## 🧭 2. Architecture & structure

### ✔️ Arborescence

- [ ] Structure logique et cohérente
- [ ] Profondeur raisonnable (≤ 3 clics pour les pages importantes)
- [ ] URLs propres, descriptives et stables

### ✔️ Pages piliers & clusters

- [ ] Pages piliers identifiées
- [ ] Clusters thématiques cohérents
- [ ] Navigation claire entre les pages d’un même cluster

---

## 🔗 3. Maillage interne

### ✔️ Liens internes

- [ ] Liens contextuels présents
- [ ] Ancres descriptives
- [ ] Pages importantes bien maillées
- [ ] Aucune page orpheline

### ✔️ Navigation

- [ ] Menu clair
- [ ] Fil d’Ariane présent si pertinent
- [ ] Liens transversaux entre pages liées

---

## 🔍 4. Indexation & crawl

### ✔️ Conditions d’indexation

- [ ] Page accessible sans interaction
- [ ] Pas de `noindex` involontaire
- [ ] Pas de blocage robots.txt
- [ ] Contenu visible par Google

### ✔️ Crawl

- [ ] Profondeur optimisée
- [ ] Maillage interne cohérent
- [ ] Pas de chaînes de redirections
- [ ] Sitemaps à jour (si nécessaire)

---

## 🧩 5. Données structurées

### ✔️ Pertinence

- [ ] Schéma adapté à la page (Article, Product, FAQ, etc.)
- [ ] JSON‑LD utilisé
- [ ] Champs obligatoires présents dans le contenu visible

### ✔️ Cohérence

- [ ] Données structurées cohérentes avec le texte
- [ ] Pas de schémas artificiels ou trompeurs
- [ ] Pas de duplication inutile

---

## 🖼️ 6. Images (SEO)

### ✔️ Sémantique

- [ ] Texte alternatif (`alt`) descriptif
- [ ] Noms de fichiers pertinents
- [ ] Images pertinentes et non génériques

### ✔️ Contexte

- [ ] Images entourées de texte pertinent
- [ ] Légendes utilisées si utile

👉 Pour les optimisations techniques → `/performance/images.md`.

---

## ⚡ 7. Signaux essentiels (non techniques)

### ✔️ UX & lisibilité

- [ ] Contenu scannable
- [ ] Titres clairs
- [ ] Mise en forme cohérente
- [ ] Ton clair et pédagogique

### ✔️ Engagement

- [ ] Contenu utile
- [ ] Structure logique
- [ ] Navigation fluide
